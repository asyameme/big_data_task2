# Task 02

Решение считает две величины:

```text
VWAP_s
VWAP_m
```

`VWAP_s` считается по секундам, `VWAP_m` считается по минутам.

Для каждого временного интервала отдельно агрегируются buy- и sell-сделки:

```text
amount = price * quantity
VWAP = sum(amount) / sum(quantity)
```

Для buy-сделок используется условие `is_buyer_maker == False`, для sell-сделок - `is_buyer_maker == True`.

Итоговая величина считается как сумма модулей разности между buy VWAP и sell VWAP:

```text
sum(abs(VWAP_buy - VWAP_sell))
```

Если в некоторой секунде или минуте нет сделок одной стороны, значение VWAP этой стороны протягивается из ближайшего предыдущего интервала, где такие сделки были.

## Ввод и вывод

Программа принимает parquet-файл с входными данными и выводит две строки.

Пример локального запуска:

```bash
python calc_data.py input.parquet
```

Пример запуска в контейнере:

```bash
docker run --rm -v "$PWD:/data:ro" calc_data_task2 /data/input.parquet
```

Формат вывода:

```text
VWAP_s=99300.996599
VWAP_m=10074.471609
```

## Формат входных данных

Ожидается parquet-файл со схемой:

```text
trade_id          uint64
timestamp         timestamp[ns]
price             double
quantity          double
quote_qty         double
is_buyer_maker    bool
```

В решении используются только колонки:

```text
timestamp
price
quantity
is_buyer_maker
```

## Алгоритм

Данные читаются батчами через Polars streaming API, чтобы не загружать весь parquet-файл в память.

Для каждого батча:

1. Для timestamp[ns] берется целочисленное epoch-представление в наносекундах.
2. Для каждой сделки считается amount = price * quantity.
3. Сделки агрегируются по секундам.
4. Сделки агрегируются по минутам.
5. Для каждого интервала считаются суммы buy_amount, buy_qty, sell_amount, sell_qty.

После обработки всех батчей частичные агрегаты объединяются. Затем вычисляются buy VWAP и sell VWAP, выполняется forward fill пропущенных значений и считается итоговая сумма.

Для финального суммирования используется алгоритм компенсационного суммирования Кэхана, чтобы уменьшить накопление ошибки при работе с числами с плавающей точкой.
